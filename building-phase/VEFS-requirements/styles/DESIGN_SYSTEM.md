# VEFS Design System

**Status:** Established
**Created:** 2025-12-23
**Last Updated:** 2025-12-23

## Overview

The VEFS Design System provides comprehensive guidelines for the visual identity and user experience across all website pages. The system emphasizes ecological consciousness, trustworthiness, and accessibility while maintaining a modern, professional aesthetic.

## 2. Typography System

### Font Families

#### Heading Font (Serif - Trust & Elegance)
**Font Name:** Georgia, Garamond, or equivalent
- **Category:** Serif
- **Use Cases:**
  - Page titles (H1)
  - Section headings (H2)
  - Large call-outs
  - Hero text
- **Weights:** Regular (400), Bold (700)
- **Characteristics:** Elegant, professional, traditional, trustworthy
- **Alternative:** If not available, use Times New Roman or equivalent serif

#### Body Font (Sans-Serif - Readability)
**Font Name:** -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, "Helvetica Neue", Arial, sans-serif (System Font Stack)
- **Category:** Sans-serif
- **Use Cases:**
  - Body text
  - Navigation
  - Buttons
  - Small text
  - Form labels
- **Weights:** Regular (400), Medium (500), Bold (700)
- **Characteristics:** Modern, clean, readable, accessible
- **Benefits:** System font = faster loading, consistent with OS

### Type Scale

**Base Font Size:** 16px (1rem)

| Element | Size | Weight | Line Height | Letter Spacing | Use Case |
|---------|------|--------|-------------|----------------|---------|
| H1 (Page Title) | 48px | 700 (Bold) | 1.2 | -1px | Main page headings |
| H2 (Section Title) | 36px | 700 (Bold) | 1.3 | -0.5px | Section headers |
| H3 (Subsection) | 28px | 700 (Bold) | 1.4 | 0px | Subsection headers |
| H4 (Minor Heading) | 24px | 700 (Bold) | 1.5 | 0px | Small headings |
| H5 (Label) | 18px | 600 (Semibold) | 1.5 | 0px | Form labels, card titles |
| H6 (Small Label) | 14px | 600 (Semibold) | 1.5 | 0.5px | Small labels |
| Body (Large) | 18px | 400 (Regular) | 1.6 | 0px | Large body text |
| Body (Regular) | 16px | 400 (Regular) | 1.6 | 0px | Standard text |
| Body (Small) | 14px | 400 (Regular) | 1.5 | 0px | Secondary text |
| Caption | 12px | 400 (Regular) | 1.4 | 0.5px | Image captions, helper text |
| Button | 16px | 600 (Semibold) | 1.5 | 0.5px | Button text |

### Typography Usage

#### Page Titles (H1)
- Serif font, 48px, bold
- Sage green (#6B8E23)
- Margin bottom: 24px
- Example: "About Us" or "Discover VEFS"

#### Section Headings (H2)
- Serif font, 36px, bold
- Sage green (#6B8E23)
- Margin top: 48px, margin bottom: 24px
- Used for major section dividers

#### Body Text
- Sans-serif font, 16px regular
- Dark gray (#5A5A5A)
- Line height: 1.6
- Margin bottom: 16px between paragraphs

#### Links
- Color: Sage green (#6B8E23)
- Text decoration: Underline on hover
- Weight: Regular

## 3. Spacing System

**Base Unit:** 8px

### Spacing Scale

| Scale | px | rem | Use Case |
|-------|----|----|----------|
| xs | 4px | 0.25rem | Tight spacing, micro interactions |
| sm | 8px | 0.5rem | Small elements, gaps between items |
| md | 16px | 1rem | Standard spacing, button padding |
| lg | 24px | 1.5rem | Section padding, major gaps |
| xl | 32px | 2rem | Large gaps, between sections |
| 2xl | 48px | 3rem | Major section breaks |
| 3xl | 64px | 4rem | Page-level spacing |
| 4xl | 96px | 6rem | Hero sections, large spacing |

### Margin & Padding Guidelines

**Button Padding:**
- Horizontal: 16px (md)
- Vertical: 12px

**Card Padding:**
- Standard: 24px (lg)
- Compact: 16px (md)

**Section Padding:**
- Desktop: 48-64px (2xl-3xl) top and bottom
- Tablet: 32-48px (xl-2xl) top and bottom
- Mobile: 24-32px (lg-xl) top and bottom

**Grid Gap:**
- Desktop: 32px (xl)
- Tablet: 24px (lg)
- Mobile: 16px (md)

## 4. Component Guidelines

### Buttons

#### Primary Button
- **Background:** Golden/Amber (#FBBF24)
- **Text Color:** Dark gray or navy (#1F2937)
- **Border:** None
- **Padding:** 12px 24px
- **Border Radius:** 4px
- **Font Weight:** 600 (Semibold)
- **Hover:** Background changes to #F59E0B, slight shadow
- **Active:** Background darker, slight inset shadow
- **Focus:** Outline with 2px sage green
- **Usage:** Primary CTAs (Donate, Register, etc.)

#### Secondary Button
- **Background:** Light gray (#F5F5F5) or transparent
- **Text Color:** Sage green (#6B8E23)
- **Border:** 1px solid sage green
- **Padding:** 12px 24px
- **Border Radius:** 4px
- **Font Weight:** 600 (Semibold)
- **Hover:** Background light gray, text darker green
- **Focus:** Outline with 2px sage green
- **Usage:** Secondary actions, alternative CTAs

#### Button Sizes
- **Large:** 16px padding vertical, 24px horizontal
- **Standard:** 12px padding vertical, 24px horizontal
- **Small:** 8px padding vertical, 16px horizontal

### Cards

#### Card Structure
- **Background:** White (#FAFAF9) or light gray (#F5F5F5)
- **Border Radius:** 8px
- **Box Shadow:** 0 2px 8px rgba(0,0,0,0.08)
- **Padding:** 24px (lg)
- **Border:** 1px solid rgba(0,0,0,0.05) (subtle)

#### Card Hover State
- Box Shadow: 0 4px 12px rgba(0,0,0,0.12)
- Slight lift effect (transform: translateY(-2px))

### Forms

#### Input Fields
- **Background:** White
- **Border:** 1px solid #E5E7EB (light gray)
- **Border Radius:** 4px
- **Padding:** 12px 16px
- **Font Size:** 16px
- **Focus State:** Border color sage green, box shadow: 0 0 0 3px rgba(107, 142, 35, 0.1)

#### Labels
- **Font Size:** 14px (small)
- **Font Weight:** 600
- **Color:** Dark gray (#5A5A5A)
- **Margin Bottom:** 8px

#### Helper Text
- **Font Size:** 12px
- **Color:** Medium gray (#9CA3AF)
- **Margin Top:** 4px

### Navigation

#### Main Navigation Bar
- **Background:** Sage green (#6B8E23)
- **Text Color:** White
- **Padding:** 16px 32px
- **Font Size:** 16px
- **Font Weight:** 500

#### Navigation Links (Active State)
- **Background:** Darker green or accent color
- **Underline:** 2px golden/amber (#FBBF24)

#### Mobile Navigation
- **Hamburger Icon:** White, 24x24px
- **Menu Overlay:** Dark overlay (rgba(0,0,0,0.5)) with menu drawer
- **Link Color:** Sage green on white background

### Sections & Containers

#### Full-Width Section
- **Max Content Width:** 1200px
- **Horizontal Padding:** 32px (xl) on desktop, 24px (lg) on tablet, 16px (md) on mobile
- **Vertical Padding:** 64px (3xl) on desktop, 48px (2xl) on tablet, 32px (lg) on mobile

#### Hero Section
- **Min Height:** 400px desktop, 300px tablet, 250px mobile
- **Background:** Image or gradient overlay
- **Text:** White with text shadow for readability

## 5. Responsive Breakpoints

| Device | Width | Columns | Padding |
|--------|-------|---------|---------|
| Mobile | < 640px | 1 | 16px |
| Tablet | 640px - 1023px | 2 | 24px |
| Desktop | 1024px+ | 3-4 | 32px |
| Large Desktop | 1440px+ | 4+ | 48px |

### Mobile-First Approach
- Design starts for mobile (320px minimum)
- Progressively enhance for larger screens
- Touch targets minimum 44x44px on mobile

## 6. Imagery & Media

### Image Guidelines

#### Hero Images
- **Aspect Ratio:** 16:9 or 21:9
- **Resolution:** 1920x1080px minimum for full-width
- **Format:** WebP (primary), JPG (fallback)
- **Compression:** Optimized for web (< 200KB)
- **Content:** Nature, trees, communities, ecological focus

#### Card Images
- **Aspect Ratio:** 4:3 or 1:1
- **Resolution:** 600x450px or 600x600px
- **Format:** WebP or JPG
- **Compression:** < 100KB

#### Thumbnails & Icons
- **Size:** 24px, 32px, 48px, 64px
- **Format:** SVG (preferred) or PNG
- **Style:** Flat, consistent line weight

### Video Guidelines
- **Format:** MP4 (H.264) or WebM
- **Aspect Ratio:** 16:9
- **Optimization:** Adaptive bitrate for different devices
- **Thumbnail:** 16:9 placeholder image

## 7. Accessibility Standards

### WCAG 2.1 Compliance Level: AA

### Color Contrast Ratios
- **Normal Text:** 4.5:1 minimum
- **Large Text (18px+):** 3:1 minimum
- **Graphics & UI Components:** 3:1 minimum

**Color Combinations to Verify:**
- Sage Green (#6B8E23) on Off-White (#FAFAF9): 5.8:1 ✓
- Golden/Amber (#FBBF24) on Navy (#1F2937): 9.5:1 ✓
- Dark Gray (#5A5A5A) on Off-White: 6.2:1 ✓

### Font Sizes
- **Minimum body text:** 14px
- **Minimum secondary text:** 12px
- **Clickable elements:** 44x44px minimum touch target

### Interactive Elements
- **Focus Indicators:** Visible focus outline (2px, sage green)
- **Keyboard Navigation:** All interactive elements accessible
- **ARIA Labels:** Form inputs, buttons, icons
- **Alt Text:** Descriptive for all images

## 8. Animation & Transitions

### Transition Timing
- **Standard:** 0.3s ease-in-out
- **Fast:** 0.15s ease-out (micro-interactions)
- **Slow:** 0.5s ease-in-out (page transitions)

### Animations
- **Fade In:** 0.3s opacity transition
- **Slide In:** 0.4s transform + opacity
- **Hover States:** 0.2s scale and shadow changes
- **Scroll Animations:** Fade-in as content appears

### Motion Preferences
- **Respect prefers-reduced-motion:** Disable animations for users with vestibular disorders
- **No auto-playing animations:** User initiates movement

## 10. Brand Voice & Tone

### Language
- **Tone:** Professional yet approachable
- **Voice:** Educational, inspiring, community-focused
- **Terminology:** Eco-friendly terms, accessibility-focused
- **Clarity:** Clear, jargon-free, inclusive

### Messaging Hierarchy
1. **Headings:** Bold, action-oriented, benefit-focused
2. **Subheadings:** Clarifying, supportive
3. **Body Text:** Detailed, informative, encouraging
4. **CTAs:** Action verbs (Donate, Register, Explore, Discover)

---

## 11. Consistency Guidelines

### Logo Usage
- **Minimum Size:** 120px width
- **Padding:** 16px clear space around logo
- **Placement:** Top-left of navigation or header
- **Variations:** Full logo (on light backgrounds), simplified (on dark)
- **Reuse:** Existing VEFS logo as approved

### Visual Consistency
- **Grid System:** 12-column grid for desktop, 4-column for mobile
- **Hierarchy:** Consistent heading sizes, spacing, emphasis
- **Patterns:** Repeated card styles, button treatments
- **Components:** Reusable across pages

---

## 12. Implementation Notes

### CSS Architecture
- Use CSS Custom Properties (CSS Variables) for colors and spacing
- Follow BEM naming convention for classes
- Implement mobile-first media queries
- Use flexbox and grid for layouts

### Performance
- Optimize all images for web (WebP format)
- Lazy load images below the fold
- Minimize CSS/JS bundling
- Use system fonts where possible

### Browser Support
- Modern browsers (Chrome, Firefox, Safari, Edge)
- Mobile browsers (iOS Safari, Chrome Mobile)
- Minimum: IE 11 (graceful degradation)
